Introdução a AJAX

AJAX (Asynchronous JavaScript and XML) é uma técnica de desenvolvimento web que
permite a atualização de partes de uma página da web sem recarregar a página
inteira. Isso melhora a experiência do usuário, pois as páginas se tornam mais
interativas e responsivas. Embora o "XML" esteja no nome, AJAX pode usar outros
formatos de dados, como JSON ou HTML, além do XML.

A principal ideia por trás do AJAX é enviar e receber dados do servidor de
forma assíncrona, o que significa que a página da web pode continuar a ser
utilizada enquanto os dados estão sendo processados. Isso é feito através de
requisições HTTP que são realizadas em segundo plano.

Requisições HTTP Assíncronas com 'fetch'

O método fetch é uma API moderna e mais simples para fazer requisições assíncronas
em comparação com XMLHttpRequest. Ele retorna uma promessa que resolve em um objeto
Response que pode ser manipulado para obter os dados retornados pelo servidor.

Aqui está um exemplo básico de como usar o fetch para fazer uma requisição GET: 

<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Exemplo AJAX com Fetch</title>
</head>
<body>
    <h1>Dados de um Usuário</h1>
    <button id="carregar-dados">Carregar Dados</button>
    <div id="dados-usuario"></div>

    <script>
        document.getElementById('carregar-dados').addEventListener('click', function() {
            fetch('https://jsonplaceholder.typicode.com/users/1')
                .then(response => {
                    if (!response.ok) {
                        throw new Error('Erro na requisição');
                    }
                    return response.json();
                })
                .then(data => {
                    const dadosDiv = document.getElementById('dados-usuario');
                    dadosDiv.innerHTML = `<p>Nome: ${data.name}</p>
                                          <p>Email: ${data.email}</p>`;
                })
                .catch(error => {
                    console.error('Erro:', error);
                });
        });
    </script>
</body>
</html>

Explicação:

  • O exemplo acima cria uma interface simples com um botão. Quando o botão é
    clicado, a função JavaScript faz uma requisição GET para a URL especificada
    (neste caso, uma API fictícia que retorna os dados de um usuário).
  • A função fetch faz a requisição e retorna uma promessa. Se a requisição for
    bem-sucedida, os dados são convertidos para JSON usando response.json().
  • Esses dados JSON são então usados para atualizar o conteúdo da página sem
    recarregá-la.

Requisições HTTP Assíncronas com 'XMLHttpRequest'

Antes do fetch, a maneira mais comum de fazer requisições AJAX era
com XMLHttpRequest. Embora seja mais verboso e complexo, ainda é amplamente utilizado
e compreendido.

Aqui está um exemplo básico de como usar XMLHttpRequest para fazer uma requisição GET: 

<DOCTYPE html>
<tml lang="pt-BR">
<ead>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Exemplo AJAX com XMLHttpRequest</title>
</head>
<body>
    <h1>Dados de um Usuário</h1>
    <button id="carregar-dados">Carregar Dados</button>
    <div id="dados-usuario"></div>

    <script>
        document.getElementById('carregar-dados').addEventListener('click', function() {
            const xhr = new XMLHttpRequest();
            xhr.open('GET', 'https://jsonplaceholder.typicode.com/users/1', true);

            xhr.onreadystatechange = function() {
                if (xhr.readyState === 4 && xhr.status === 200) {
                    const data = JSON.parse(xhr.responseText);
                    const dadosDiv = document.getElementById('dados-usuario');
                    dadosDiv.innerHTML = `<p>Nome: ${data.name}</p>
                                          <p>Email: ${data.email}</p>`;
                } else if (xhr.readyState === 4 && xhr.status !== 200) {
                    console.error('Erro na requisição');
                }
            };

            xhr.send();
        });
    </script>
</body>
</html>

Explicação:

  • XMLHttpRequest é um objeto que permite a realização de requisições HTTP. O
    método open configura a requisição, especificando o tipo de requisição (GET)
    e a URL.
  • A propriedade onreadystatechange é um evento que é chamado sempre que o estado
    da requisição muda. O estado é verificado por xhr.readyState, onde 4 significa
    que a operação está completa.
  • Se a requisição for bem-sucedida (status 200), os dados são processados e a
    página é atualizada com as informações do usuário.

Comparação entre 'fetch' e 'XMLHttpRequest'

  • Simplicidade: fetch é muito mais simples e menos verboso, tornando o código
    mais legível e fácil de manter.
  • Suporte a Promises: fetch usa Promises, o que facilita o manuseio de
    requisições assíncronas e erros. XMLHttpRequest requer callbacks, o que
    pode levar a um código mais complexo.
  • Mais Moderno: fetch é a API recomendada para novos projetos, enquanto
    XMLHttpRequest ainda é suportado para compatibilidade com navegadores mais
    antigos.

Conclusão

AJAX é uma técnica poderosa que permite melhorar a interatividade das páginas web,
fazendo com que as requisições ao servidor sejam feitas em segundo plano. Tanto
'fetch' quanto 'XMLHttpRequest' permitem a realização de requisições assíncronas, mas
'fetch' é geralmente preferido devido à sua simplicidade e ao suporte a Promises.
