Gestão de Erros em JavaScript

Erros em JavaScript podem ocorrer por diversos motivos, como referências a
variáveis inexistentes, erros de sintaxe, ou problemas de lógica. Gerenciar
esses erros de forma eficaz é essencial para garantir que seu aplicativo
funcione corretamente e ofereça uma boa experiência ao usuário.

  1. Blocos 'try...catch'

    2. O bloco try...catch é uma das maneiras mais comuns de lidar com erros em JavaScript.
    Ele permite que você execute um bloco de código e capture qualquer exceção que
    ocorra, para que você possa tratá-la de forma controlada.

    Exemplo:

    try {
    // Código que pode gerar um erro
    let resultado = divisao(10, 0);
    console.log(resultado);
} catch (erro) {
    // Tratamento do erro
    console.error('Ocorreu um erro: ', erro.message);
}

function divisao(a, b) {
    if (b === 0) {
        throw new Error('Divisão por zero não permitida');
    }
    return a / b;
}

    Neste exemplo, o bloco 'try' tenta executar uma divisão. Se o divisor for zero, um
    erro é lançado ('throw new Error'). O bloco 'catch' captura esse erro e exibe uma
    mensagem amigável.

  2. Tratamento Global de Erros

    3. Em aplicações maiores, pode ser útil configurar um tratamento global de erros,
    para capturar exceções que não foram tratadas em outros lugares do código.

    Exemplo com 'window.onerror': 

    window.onerror = function(mensagem, fonte, linha, coluna, erro) {
    console.error(`Erro: ${mensagem} em ${fonte}:${linha}:${coluna}`);
    // Aqui você pode enviar os detalhes do erro para um servidor para análise posterior
    return true; // Impede que o erro seja exibido no console padrão do navegador
};

    Exemplo com 'window.addEventListener':

    window.addEventListener('error', function(evento) {
    console.error('Erro capturado:', evento.message);
    // Você pode enviar os detalhes do erro para um servidor aqui
});

    Esses métodos permitem capturar erros em nível global, o que pode ser útil para
    logar erros em produção e monitorar problemas que os usuários enfrentam.

  3. Exceções Personalizadas

    4. Você pode criar exceções personalizadas para situações específicas, melhorando
    a clareza do código e facilitando o tratamento de erros.

    Exemplo: 

        class ValidacaoErro extends Error {
        constructor(mensagem) {
            super(mensagem);
            this.name = 'ValidacaoErro';
        }
    }
    
    function validarEntrada(idade) {
        if (idade < 18) {
            throw new ValidacaoErro('A idade deve ser maior que 18 anos.');
        }
        return true;
    }
    
    try {
        validarEntrada(15);
    } catch (erro) {
        if (erro instanceof ValidacaoErro) {
            console.warn(erro.message);
        } else {
            throw erro; // Rethrow if it's an unknown error
        }
    }

    Neste exemplo, ValidacaoErro é uma exceção personalizada que é lançada se a
    entrada não atender aos critérios definidos.

Ferramentas de Debugging no Navegador

Os navegadores modernos vêm com ferramentas de desenvolvimento (DevTools) que
facilitam o processo de debugging. Abaixo estão algumas das funcionalidades mais
úteis:

  1. Console do JavaScript

    2. O console é um dos recursos mais usados para debugging. Ele permite que você
    visualize logs, erros, e até execute JavaScript diretamente no contexto da página.

    Exemplo:

    console.log('Mensagem informativa');
console.warn('Aviso de algo que pode não estar certo');
console.error('Mensagem de erro');
  2. Breakpoints

    3. Breakpoints permitem que você pause a execução do código em pontos específicos
    para inspecionar o estado das variáveis e entender o fluxo do programa.

    • Adicionando um Breakpoint: Clique na linha de código desejada no painel
      "Sources" das DevTools para adicionar um breakpoint.
    • Exemplo Prático:
      • function somar(a, b) {
    return a + b; // Adicione um breakpoint aqui
}

let resultado = somar(2, 3);
console.log('Resultado:', resultado);

      Quando o código atinge o breakpoint, a execução para, e você pode
      inspecionar as variáveis 'a' e 'b' para ver seus valores.

  3. Watch Expressions

    4. Watch Expressions permitem que você monitore o valor de expressões
    JavaScript enquanto o código é executado. Isso é útil para rastrear como
    os valores mudam ao longo do tempo.

    Exemplo:

        function calcularArea(raio) {
        let area = Math.PI * raio * raio;
        return area;
    }
    
    let resultado = calcularArea(5);
    console.log('Área:', resultado);

    Adicione uma expressão como 'a + b' para monitorar diretamente como o
    resultado muda durante a execução do código.

  4. Call Stack

    5. A Call Stack mostra a sequência de chamadas de função que levaram ao ponto
    atual de execução. Isso ajuda a entender como o código chegou a um determinado estado. 

    function a() {
    b();
}

function b() {
    c();
}

function c() {
    console.log('Estamos na função c'); // Veja a call stack aqui
}

a();

    Quando a função 'c' é chamada, a call stack mostrará a sequência de chamadas
    'a -> b -> c', permitindo que você rastreie o caminho de execução.

  5. Ferramenta de Network

    6. O painel "Network" permite inspecionar todas as requisições HTTP feitas pela
    página, incluindo requisições AJAX. Você pode ver detalhes como status,
    cabeçalhos, e conteúdo da resposta.

    • Abra o painel "Network".
    • Realize uma requisição AJAX na página.
    • Inspecione a requisição para ver detalhes como tempo de resposta,
      conteúdo e possíveis erros.

    Com essas práticas e ferramentas, você estará mais bem preparado para lidar
    com erros e fazer debugging em JavaScript, garantindo que seus projetos
    sejam robustos e funcionais.
