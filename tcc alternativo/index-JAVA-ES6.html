ES6 e Recursos Modernos

ES6, ou ECMAScript 2015, trouxe melhorias significativas para o JavaScript. Recursos
como 'const' e 'let' aprimoram a declaração de variáveis, enquanto arrow functions
oferecem uma sintaxe mais concisa. Destructuring facilita a extração de valores, e
Promises melhoram a gestão de operações assíncronas, tornando o desenvolvimento mais
eficiente e organizado.

Const e Let

Em ES6, const e let foram introduzidos como novas formas de declarar variáveis.
Ambos têm escopo de bloco, o que significa que estão disponíveis apenas dentro
do bloco onde foram definidos, diferentemente do var, que tem escopo de função.

  • Const

    • Utilizado para declarar variáveis que não podem ser reatribuídas após a inicialização. 

    const pi = 3.14159;
console.log(pi); // 3.14159

// Tentar reatribuir uma constante resultará em um erro
// pi = 3.14; // Uncaught TypeError: Assignment to constant variable.
  • Let

    • Utilizado para variáveis que podem ser reatribuídas, mas que precisam de escopo de bloco. 

    let count = 0;
count++;
console.log(count); // 1

Arrow Functions

Arrow functions são uma forma mais concisa de escrever funções em JavaScript. Além
disso, elas não têm o próprio contexto 'this', o que pode ser útil ao trabalhar com
métodos de classe ou callbacks.

  • Sintaxe Concisa:
    • // Função tradicional
function add(a, b) {
    return a + b;
}

// Arrow function equivalente
const add = (a, b) => a + b;

console.log(add(2, 3)); // 5
  • Uso de Arrow Function em Callbacks:
    • const numbers = [1, 2, 3, 4, 5];
const squares = numbers.map(num => num * num);
console.log(squares); // [1, 4, 9, 16, 25]

Destructuring

Destructuring é uma maneira de extrair valores de arrays ou objetos
em variáveis distintas de forma mais simples e legível.

  • Destructuring em Arrays:
    • const colors = ['red', 'green', 'blue'];
const [firstColor, secondColor] = colors;

console.log(firstColor); // 'red'
console.log(secondColor); // 'green'
  • Destructuring em Objetos:
    • const person = {
name: 'John',
age: 30,
city: 'New York'
};
                            
const { name, age } = person;
console.log(name); // 'John'
console.log(age);  // 30

Promises

As Promises são usadas para lidar com operações assíncronas em JavaScript. Elas
representam um valor que pode estar disponível agora, no futuro ou nunca.

  • Criando e Usando uma Promise:
    • const fetchData = () => {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            const data = { id: 1, name: 'Sample Data' };
            resolve(data);
        }, 2000);
    });
};

fetchData()
    .then(data => {
        console.log(data); // { id: 1, name: 'Sample Data' }
    })
    .catch(error => {
        console.error('Error:', error);
    });
  • Usando Promises com async/await:
    • const fetchData = async () => {
    try {
        const data = await fetchData();
        console.log(data); // { id: 1, name: 'Sample Data' }
    } catch (error) {
        console.error('Error:', error);
    }
};

fetchData();
